4,5. Определите разницу между контейнером и образом

Образ — это статичный шаблон, содержащий все необходимые зависимости, файлы и настройки для запуска приложения. 
Образ включает в себя операционную систему, библиотеки, кэшированные слои приложений и может быть перенесён на любую систему, где установлен Docker.
Образ является неизменяемым и не содержит состояния, это как «снимок» или шаблон для создания контейнеров.

Контейнер — это запущенный экземпляр Docker образа.
Контейнер представляет собой процесс, который изолирован от остальной системы, использует слои образа, но может иметь своё собственное состояние и изменения в файловой системе, пока он работает.
В контейнер можно вносить изменения, которые будут существовать только до завершения его работы.

6. Можно ли в контейнере собрать ядро?

Да, в контейнере можно собрать Linux ядро
Главное условие, это чтобы контейнер был запущен в привилегированном режиме. Также должны быть установлены необходимые зависимости (gcc, make и т.д.) и
проброшен доступ до ресурсов хоста, необходимых для сборки.
